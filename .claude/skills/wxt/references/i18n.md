# 国际化 (I18n)

WXT 提供了强大的国际化 (i18n) 支持，使扩展可以轻松支持多种语言。

---

## 基本概念

WXT 的 i18n 系统基于浏览器扩展的标准 i18n 机制：

- 使用 `_locales` 目录存放翻译文件
- 使用 `__MSG_***__` 占位符在 manifest 中引用翻译
- 使用 `browser.i18n.getMessage()` 在代码中获取翻译

---

## 目录结构

```
📂 {rootDir}/
   📁 public/_locales/
      📂 en/
         📄 messages.json
      📂 zh_CN/
         📄 messages.json
      📂 ja/
         📄 messages.json
```

**重要**：`_locales` 目录必须放在 `public/` 目录中，因为浏览器需要从扩展根目录访问它。

## 创建翻译文件

### messages.json 格式

每个 `messages.json` 文件包含键值对，值包含 `message` 和可选的 `description`：

```json
{
  "extensionName": {
    "message": "My Extension",
    "description": "The name of the extension"
  },
  "extensionDescription": {
    "message": "A great browser extension",
    "description": "The description of the extension"
  },
  "btnTitle": {
    "message": "Click me",
    "description": "Button title"
  }
}
```

## 在 Manifest 中使用

### 配置默认语言

在 `wxt.config.ts` 中设置：

```typescript
export default defineConfig({
  manifest: {
    default_locale: 'en',
  },
});
```

### 使用占位符

在 manifest 中使用 `__MSG_***__` 占位符：

```typescript
export default defineConfig({
  manifest: {
    name: '__MSG_extensionName__',
    description: '__MSG_extensionDescription__',
    default_locale: 'en',
  },
});
```

生成的 `manifest.json`：
```json
{
  "name": "__MSG_extensionName__",
  "description": "__MSG_extensionDescription__",
  "default_locale": "en"
}
```

## 在代码中使用

### 获取翻译

使用 `browser.i18n.getMessage()` 获取翻译：

```typescript
import { browser } from 'wxt/browser';

export default defineBackground(() => {
  const extensionName = browser.i18n.getMessage('extensionName');
  console.log(extensionName); // "My Extension" (或对应语言的翻译)
});
```

### 在 Content Script 中

```typescript
export default defineContentScript({
  matches: ['<all_urls>'],

  main() {
    const btnTitle = browser.i18n.getMessage('btnTitle');

    const button = document.createElement('button');
    button.textContent = btnTitle;
    document.body.appendChild(button);
  },
});
```

### 在 Vue 组件中

```vue
<script setup lang="ts">
import { browser } from 'wxt/browser';
import { computed } from 'vue';

const title = computed(() => browser.i18n.getMessage('btnTitle'));
</script>

<template>
  <button>{{ title }}</button>
</template>
```

### 在 React 组件中

```tsx
import { browser } from 'wxt/browser';

function Button() {
  const title = browser.i18n.getMessage('btnTitle');
  return <button>{title}</button>;
}
```

## 带占位符的翻译

### 定义占位符

在 `messages.json` 中使用 `$1$`, `$2$` 等占位符：

```json
{
  "greeting": {
    "message": "Hello, $1$! Today is $2$.",
    "description": "Greeting message with placeholders"
  },
  "itemCount": {
    "message": "You have $1$ items",
    "description": "Item count message"
  }
}
```

### 使用占位符

```typescript
// 简单替换
const greeting = browser.i18n.getMessage('greeting', 'Alice');
// "Hello, Alice! Today is $2$."

// 多个占位符
const greeting = browser.i18n.getMessage('greeting', ['Alice', 'Monday']);
// "Hello, Alice! Today is Monday."

// 数字替换
const count = browser.i18n.getMessage('itemCount', '5');
// "You have 5 items"
```

## 语言标签

### BCP 47 语言标签

翻译目录名必须使用有效的 BCP 47 语言标签：

| 语言 | 标签 | 目录名 |
|------|------|--------|
| 英语（通用） | `en` | `en/` |
| 简体中文 | `zh-CN` | `zh_CN/` (目录名使用下划线) |
| 繁体中文 | `zh-TW` | `zh_TW/` |
| 日语 | `ja` | `ja/` |
| 韩语 | `ko` | `ko/` |
| 法语 | `fr` | `fr/` |
| 德语 | `de` | `de/` |
| 西班牙语 | `es` | `es/` |
| 俄语 | `ru` | `ru/` |
| 葡萄牙语（巴西） | `pt-BR` | `pt_BR/` |

**注意**：虽然 BCP 47 标签使用连字符（`zh-CN`），但目录名必须使用下划线（`zh_CN`）。

## 检测用户语言

### 获取浏览器语言

```typescript
import { browser } from 'wxt/browser';

export default defineBackground(() => {
  // 获取浏览器 UI 语言
  const uiLanguage = browser.i18n.getUILanguage();
  console.log(uiLanguage); // "zh-CN", "en", "ja", etc.

  // 获取接受的语言列表
  browser.i18n.getAcceptLanguages((languages) => {
    console.log(languages); // ["zh-CN", "zh", "en-US", "en"]
  });
});
```

### 根据语言执行逻辑

```typescript
export default defineBackground(() => {
  const language = browser.i18n.getUILanguage();

  if (language.startsWith('zh')) {
    // 中文用户特定逻辑
    console.log('Chinese user');
  } else if (language.startsWith('ja')) {
    // 日语用户特定逻辑
    console.log('Japanese user');
  } else {
    // 默认逻辑
    console.log('Other user');
  }
});
```

## 自动类型定义

### 创建类型文件

为了获得更好的 TypeScript 支持，可以创建类型定义：

**types/i18n.d.ts**：
```typescript
declare module 'wxt/browser' {
  export interface BrowserI18n {
    getMessage(messageName: 'extensionName'): string;
    getMessage(messageName: 'extensionDescription'): string;
    getMessage(messageName: 'btnTitle'): string;
    getMessage(messageName: 'greeting', substitutions?: string | string[]): string;
    getMessage(messageName: 'itemCount', substitutions?: string | string[]): string;
  }
}
```

### 使用类型安全

```typescript
// TypeScript 现在知道这些消息名称
const name = browser.i18n.getMessage('extensionName');
const greeting = browser.i18n.getMessage('greeting', ['Alice', 'Monday']);
```

**public/_locales/zh_CN/messages.json**：

### wxt.config.ts

```typescript
import { defineConfig } from 'wxt';

export default defineConfig({
  manifest: {
    name: '__MSG_extensionName__',
    description: '__MSG_extensionDescription__',
    default_locale: 'en',
  },
});
```

### Background Script

```typescript
import { browser } from 'wxt/browser';

export default defineBackground(() => {
  // 获取扩展名称
  const name = browser.i18n.getMessage('extensionName');
  console.log('Extension:', name);

  // 安装时显示欢迎消息
  browser.runtime.onInstalled.addListener(async () => {
    const extensionName = browser.i18n.getMessage('extensionName');
    const welcomeTitle = browser.i18n.getMessage('welcomeTitle');
    const welcomeMessage = browser.i18n.getMessage('welcomeMessage', extensionName);

    // 可以显示通知或保存到 storage
    await browser.storage.local.set({
      welcomeMessage: `${welcomeTitle}: ${welcomeMessage}`,
    });
  });
});
```

### Popup (Vue)

```vue
<script setup lang="ts">
import { browser } from 'wxt/browser';
import { ref, onMounted } from 'vue';

const title = ref('');
const message = ref('');
const settingsTitle = ref('');

onMounted(() => {
  title.value = browser.i18n.getMessage('welcomeTitle');
  message.value = browser.i18n.getMessage('welcomeMessage', ['My Extension']);
  settingsTitle.value = browser.i18n.getMessage('settingsTitle');
});
</script>

<template>
  <div class="popup">
    <h1>{{ title }}</h1>
    <p>{{ message }}</p>
    <button>{{ settingsTitle }}</button>
  </div>
</template>

<style scoped>
.popup {
  padding: 16px;
  width: 300px;
}
</style>
```

### Content Script

```typescript
export default defineContentScript({
  matches: ['<all_urls>'],

  main() {
    // 创建带有翻译文本的按钮
    const settingsTitle = browser.i18n.getMessage('settingsTitle');

    button.style.top = '10px';
    button.style.right = '10px';
    button.style.zIndex = '999999';

    document.body.appendChild(button);
  },
});
```

## 最佳实践

### 1. 提供默认语言

始终提供英语作为默认语言：

```
📁 public/_locales/
   📂 en/        ← 默认语言
      📄 messages.json
```

### 2. 提供完整的描述

为所有翻译键提供 `description`：

```json
{
  "btnTitle": {
    "message": "Click me",
    "description": "Title for the main action button" // 有助于翻译者理解上下文
  }
}
```

### 3. 使用一致的键名

使用清晰、一致的键名：

```json
// 推荐
{
  "settingsTitle": "...",
  "settingsDescription": "...",
  "settingsButton": "..."
}

// 避免
{
  "title1": "...",
  "desc1": "...",
  "btn1": "..."
}
```

### 4. 避免硬编码文本

始终使用 i18n 而不是硬编码文本：

```typescript
// 不推荐
const title = 'Settings';

// 推荐
const title = browser.i18n.getMessage('settingsTitle');
```

### 5. 测试所有语言

在开发和测试时切换浏览器语言，确保所有翻译正确显示。
